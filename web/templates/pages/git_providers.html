How Git providers work

  • Create a reusable provider once, then select it from each app Git tab.
  • GitHub App flow is the preferred path because NextDeploy can mint short-lived installation tokens and auto-create repo webhooks.
  • GitLab remains supported for private repo cloning, but webhook setup stays manual for now.
  • Legacy token fields remain available only for compatibility with older setups.

GitHub setup

Dokploy-style GitHub App setup. NextDeploy prefills the registration flow and stores the returned app credentials automatically.

After GitHub setup

Suggested GitHub App permissions

Repository contents: read-only, metadata: read-only, administration: write, webhook event: push. Administration is required so NextDeploy can create or update the repository webhook automatically.
